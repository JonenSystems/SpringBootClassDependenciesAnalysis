SpringBoot依存関係解析機能要求一覧

■ 型依存の解析機能
REQ-001 要求 解析対象のクラス、メソッド、フィールドが型に関する依存関係が存在する時、型依存に関する依存関係情報を収集する。
理由 - Javaの型システムにおける基本的な依存関係を把握するため

- クラス間の継承関係や実装関係を可視化するため

- パッケージ境界を越えた型参照の影響範囲を特定するため
説明 - 用語: 型依存とは、クラス、メソッド、フィールドが他の型（クラス、インターフェース）に依存する関係

- 範囲: extends句、implements句、import文、ジェネリクス型、例外型の依存関係

- 例: class OrderService extends BaseService、class UserService implements UserDetailsService
■ 継承(extends)依存の解析
REQ-001.001 要求 解析対象のクラスがextends句/継承階層を検出した時、継承に関する依存関係情報を収集する。
理由 - クラス継承による機能引き継ぎの依存関係を把握するため

- 継承階層の循環依存を検出するため

- 親クラスの変更が子クラスに与える影響を特定するため
説明 - 用語: 継承依存とは、クラスが親クラスを継承して機能を引き継ぐ関係

- 範囲: extends句によるクラス継承関係

- 例: class OrderService extends BaseService
■ 継承依存解析仕様
SPC-001.001-001 [前提]  Javaのクラス定義を解析対象とする
[入力]  extends句を含むクラス定義
[処理]  extends句を解析し、親クラスとの依存関係を抽出
[出力] 子クラス→親クラスの依存関係情報（クラス名、パッケージ名、継承階層レベル）
[制約] 多重継承は検出対象外（Javaでは不可）
■ 実装(implements)依存の解析
REQ-001.002 要求 解析対象のクラスがimplements句/実装関係を検出した時、実装に関する依存関係情報を収集する。
理由 - インターフェース実装による依存関係を把握するため

- インターフェースの差し替え可能性を評価するため

- 実装クラスとインターフェースの結合度を測定するため
説明 - 用語: 実装依存とは、クラスがインターフェースを実装する関係

- 範囲: implements句によるインターフェース実装関係

- 例: class UserService implements UserDetailsService
■ 実装依存解析仕様
SPC-001.002-001 [前提]  Javaのクラス定義を解析対象とする
[入力]  implements句を含むクラス定義
[処理]  implements句を解析し、インターフェースとの依存関係を抽出
[出力] 実装クラス→インターフェースの依存関係情報（クラス名、インターフェース名、パッケージ名）
[制約] 複数インターフェース実装に対応する
■ 型参照(import/qualified)依存の解析
REQ-001.003 要求 解析対象のクラス、メソッド、フィールドがimport文/修飾名参照を検出した時、型参照に関する依存関係情報を収集する。
理由 - パッケージ境界を越えた型参照の依存関係を把握するため

- 他クラスを型として参照する関係を可視化するため

- パッケージ間の結合度を測定するため
説明 - 用語: 型参照依存とは、他クラスを型として参照する関係

- 範囲: import文による型インポート、修飾名による型参照

- 例: import com.example.User; User u;
■ 型参照依存解析仕様
SPC-001.003-001 [前提]  Javaのソースコードを解析対象とする
[入力]  import文および型参照を含むコード
[処理]  import文と型参照を解析し、型依存関係を抽出
[出力] 参照元→参照先の型依存関係情報（パッケージ名、クラス名、参照箇所）
[制約] 同一パッケージ内の型参照も対象とする
■ ジェネリクス型参照依存の解析
REQ-001.004 要求 解析対象のメソッド、フィールドがParameterizedType/TypeArgumentを検出した時、ジェネリクス型参照に関する依存関係情報を収集する。
理由 - ジェネリック型実引数による依存関係を把握するため

- コレクション経由の型依存を特定するため

- 型安全性の観点から依存関係を分析するため
説明 - 用語: ジェネリクス型参照依存とは、ジェネリック型実引数による依存関係

- 範囲: ParameterizedType、TypeArgumentによる型依存

- 例: List<User>, Map<Long, Order>
■ ジェネリクス型参照依存解析仕様
SPC-001.004-001 [前提]  Javaのジェネリクス構文を解析対象とする
[入力]  ParameterizedType/TypeArgumentを含むメソッド・フィールド定義
[処理]  ジェネリクス型引数を解析し、型依存関係を抽出
[出力] ジェネリクス型依存関係情報（型引数、参照箇所、ネストレベル）
[制約] ネストしたジェネリクス型に対応する
■ 例外型依存の解析
REQ-001.005 要求 解析対象のメソッドがthrows句/catch(CustomException)を検出した時、例外型に関する依存関係情報を収集する。

- 例外設計の境界を特定するため

- 範囲: throws句、catch句による例外型参照

- 例: void f() throws CustomException

SPC-001.005-001 [前提]  Javaのメソッド定義と例外処理を解析対象とする
